## 2. Назначение КИМ

КИМ предназначен для проверки того, что обучающийся понимает фундаментальные концепции предметной области, умеет формулировать задачу рекомендаций, определять тип обратной связи, готовить данные для анализа, строить простые и эффективные baseline-модели, а главное — критически оценивать данные и выявлять присущие им систематические смещения (popularity bias, фильтрующие пузыри), которые влияют на качество будущих рекомендаций.

## 7. Ход выполнения

### Часть 1. Постановка задачи
1.  Сформулируйте, какую именно задачу вы решаете: предсказание рейтинга, ранжирование или Top-N рекомендаций.
2.  Определите и опишите:
    - кто является **пользователями**;
    - что является **объектами**;
    - какой тип обратной связи доступен: **explicit** (оценки) или **implicit** (просмотры, клики, покупки).
3.  Обоснуйте свой выбор.

### Часть 2. Подготовка данных и EDA
1.  Загрузите датасет.
2.  Проверьте структуру данных, типы, наличие пропусков и дубликатов.
3.  Постройте базовые распределения:
    - распределение оценок (если explicit);
    - распределение активности пользователей (сколько оценок поставил каждый пользователь);
    - распределение популярности объектов (сколько оценок получил каждый объект).
4.  Постройте матрицу взаимодействий `user × item`.
5.  Рассчитайте разреженность матрицы:

```python
sparsity = 1 - matrix.nnz / (matrix.shape[0] * matrix.shape[1])
```

6.  Визуализируйте матрицу (например, с помощью `matplotlib.spy` или тепловой карты для сэмпла данных).

### Часть 3. Анализ смещений (Biases)
1.  Постройте **распределение популярности объектов** и **распределение активности пользователей**.
2.  Оцените влияние **popularity bias**: как много взаимодействий приходится на небольшую долю самых популярных объектов. Рассчитайте, например, долю взаимодействий топ-1%, топ-10% объектов.
3.  Обсудите, как это может повлиять на будущие рекомендации.
4.  Подумайте о **фильтрующих пузырях**. Могут ли рекомендательные системы, основанные только на популярности, создать этот эффект? Аргументируйте.
5.  Проанализируйте, как **временная метка** влияет на распределение данных и какие ограничения накладывает.

### Часть 4. Baseline-модели
Реализуйте и проанализируйте несколько простых моделей (baselines):

1.  **Global Mean**: прогнозирование глобальной средней оценки для всех пользователей и объектов.
2.  **User Mean**: прогнозирование средней оценки конкретного пользователя.
3.  **Item Mean**: прогнозирование средней оценки конкретного объекта.
4.  **Popularity-based / Top-Popular**: рекомендация самых популярных объектов для всех пользователей.
5.  Для каждой модели рассчитайте простую метрику качества на тестовой выборке (например, RMSE для рейтингов или Precision/Recall для Top-N).

### Часть 5. Анализ ограничений данных
На основе проведённого анализа ответьте на вопросы:

1.  Что такое **cold start** в контексте ваших данных? Какие проблемы он создаёт для новых пользователей и новых объектов?
2.  Как **разреженность** влияет на качество потенциальных рекомендаций?
3.  Почему **наблюдаемая обратная связь** является ограниченной? (Помните, что отсутствие оценки может означать не "не нравится", а "не видел".)
